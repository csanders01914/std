Component Breakdown

Modules, responsibilities, and boundaries

IdentityManager
generated once on first launch
Generates Ed25519 key pair
Derives Tor v3 .onion address from public key
Holds private key in memory only
Exports public key as QR code for sharing
ContactStore
in-memory only, lost on app close
Stores contact public keys + .onion addresses
Added via QR scan only
No names, metadata, or timestamps
Cleared entirely on process death
SignalSessionManager
wraps libsignal-android
Runs X3DH key agreement on first contact
Maintains Double Ratchet state per contact
Encrypts outbound messages
Decrypts inbound messages
All state in memory only
Provides forward secrecy + break-in recovery
TorTransport
wraps tor-android (Guardian Project)
Starts embedded Tor daemon on launch
Hosts a v3 hidden service to receive messages
Routes outbound connections over Tor circuit
Exposes simple send(onion, bytes) / receive callback
MessageQueue
ephemeral delivery buffer
Buffers outbound messages when recipient is offline
Retries delivery until ACK received or app closes
Drops all undelivered messages on app close
No persistence, no server-side queuing
UI (Jetpack Compose)
thin shell, no business logic
Contact list screen
Chat screen
QR scanner / generator
