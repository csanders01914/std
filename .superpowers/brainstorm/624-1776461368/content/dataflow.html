Flow 2 — Sending a Message
1
Alice types message — plaintext exists only in UI layer memory
2
SignalSessionManager encrypts via Double Ratchet → produces opaque ciphertext blob. Ratchet advances; this key is now gone forever.
3
TorTransport opens circuit to Bob's .onion address — 3 hops through Tor network. Alice's IP not visible to Bob or any relay.
4
Ciphertext delivered to Bob's hidden service. If Bob is offline → MessageQueue retries. No server stores the message.
5
Bob's TorTransport receives blob → passes to SignalSessionManager → Double Ratchet decrypts → plaintext passed to UI.
6
ACK sent back over same circuit. Alice's MessageQueue drops the buffered message. Plaintext never written to disk on either device.
What an Attacker Sees at Each Layer
Network observer
Sees encrypted Tor traffic. Cannot identify sender, recipient, or content.
Tor relay node
Sees one hop of the circuit. Never knows both ends simultaneously.
Seized device
No messages on disk. Keys in RAM only — cleared on reboot/app close.
Compromised contact
Sees only their own conversation. Cannot impersonate sender (keys verify).
